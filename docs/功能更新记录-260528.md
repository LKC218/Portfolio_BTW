### 1. 音频播放器组件重构 (`AudioPlayer.tsx`)

**核心改进**:
- **隐藏功能**: 支持完全隐藏播放器，通过Esc键或点击按钮切换
- **键盘快捷键**: 
  - `Space`: 播放/暂停切换
  - `M`: 静音/取消静音
  - `Esc`: 隐藏播放器
- **自动收起**: 展开状态3秒无操作自动收起
- **点击外部**: 点击播放器外部区域自动收起
- **错误处理**: 完善的音频加载失败提示与状态显示

**技术实现**:
```typescript
// 隐藏状态管理
const [isHidden, setIsHidden] = useState(false);

// 键盘快捷键监听
useEffect(() => {
  const handleKeyDown = (e: KeyboardEvent) => {
    if (e.code === 'Space') togglePlay();
    if (e.code === 'Escape') setIsHidden(true);
    if (e.code === 'KeyM') setVolume(prev => prev > 0 ? 0 : BGM_CONFIG.volume);
  };
  document.addEventListener('keydown', handleKeyDown);
  return () => document.removeEventListener('keydown', handleKeyDown);
}, [togglePlay]);
```

### 6. 3D模型查看器 (`ModelViewer.tsx`)

**新增组件**:
- **Three.js引擎**: 基于 `@react-three/fiber` 的声明式3D渲染
- **占位几何体**: 十二面体 + 扭曲材质 + 线框外壳 + 环绕光环
- **交互控制**: OrbitControls（左键旋转、滚轮缩放）
- **HUD叠加层**: 实时旋转角度、状态信息、操作提示
- **环境光照**: ambientLight + directionalLight + pointLight（accent color）

**技术实现**:
```typescript
interface ModelViewerProps {
  accentColor: string;   // 主题色（影响几何体、灯光、HUD）
  className?: string;     // 容器额外样式
  showHud?: boolean;      // 是否显示 HUD 叠加层
  hudPadding?: string;    // HUD 内边距（避免与外部 UI 重叠）
}
```

**场景构成**:
```typescript
// 占位几何体
<mesh ref={meshRef}>
  <dodecahedronGeometry args={[1.5, 1]} />
  <MeshDistortMaterial
    color={accentColor}
    roughness={0.2}
    metalness={0.8}
    distort={0.15}
    speed={2}
    envMapIntensity={0.5}
  />
</mesh>

// 线框外壳
<mesh ref={wireRef}>
  <dodecahedronGeometry args={[1.65, 1]} />
  <meshBasicMaterial color={accentColor} wireframe opacity={0.15} transparent />
</mesh>
```

```typescript
import { createPortal } from 'react-dom';

// 展开态模态框（z-[100]）
```

**Vite配置修改** (`vite.config.ts`):
```typescript
server: {

```

**启动脚本** (`启动.bat`):
```batch

    npm run start
) else (
    echo 启动开发服务器，端口: %~1
    npm run start -- --port %~1
)

pause
```

**使用方式**:
```bash

# 指定端口启动
```

### 2026-06-01 技术要点

### 4. React Portal 堆叠上下文修复

**问题**: SceneViewer 使用 `fixed inset-0 z-20` 创建了堆叠上下文，子组件中的模态框（`z-[100]`、`z-[200]`）无法突破父级 z-index

**解决方案**: 使用 `createPortal` 将模态框渲染到 `document.body`

```typescript
import { createPortal } from 'react-dom';

// 逃离 SceneViewer 的 z-20 堆叠上下文
{isExpanded && createPortal(
  <div className="fixed inset-0 z-[100]">...</div>,
  document.body
)}
```

**技术栈**: `@react-three/fiber` + `@react-three/drei`

- `OrbitControls`: 鼠标交互控制
- `Environment`: 环境贴图提供反射
- `Float`: 浮动动画效果

### 7. 端口配置与环境变量

**问题**: 多项目开发时端口3000冲突，需要灵活配置端口

**解决方案**: 使用环境变量配置端口，支持命令行参数覆盖

**技术实现**:
```typescript
// vite.config.ts - 从环境变量读取端口
import { loadEnv } from 'vite';

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, '.', '');
  return {
    server: {
      port: Number(env.PORT) || 3000,  // 环境变量优先，fallback到3000
      host: '0.0.0.0',
      open: true,
    },
  };
});
```

**环境变量加载机制**:
1. Vite 的 `loadEnv` 函数自动加载 `.env` 文件
2. 环境变量优先级：命令行参数 > `.env` 文件 > 默认值
3. 支持 `.env.local`、`.env.[mode]` 等多环境配置

**批处理脚本参数传递**:
```batch
@echo off
cd /d "%~dp0"

if "%~1"=="" (
    echo 启动开发服务器，使用默认端口...
    npm run start
) else (
    echo 启动开发服务器，端口: %~1
    npm run start -- --port %~1
)

pause
```

**使用示例**:
```bash
# 默认端口（从.env读取）
启动.bat

# 命令行覆盖端口
启动.bat 3001

# npm直接指定端口
npm run dev -- --port 8080
```
