# 颜色语言文档

> 本文档根据当前页面实现整理，用于统一 Portfolio_BTW 的色彩语义、组件用色规则与后续扩展口径。

---

## 1. 色彩定位

当前页面的颜色语言是：

**黑底高对比 + 酸性荧光强调 + 数据界面冷光 + 场景主题动态色**

整体风格偏向赛博档案、环境艺术作品集、工业监控界面。颜色不是单纯装饰，而承担以下功能：

1. **空间基底**：深黑、炭黑用于建立沉浸式暗场。
2. **信息层级**：白色、灰色用于标题、正文、辅助信息。
3. **交互反馈**：酸性绿、电蓝、警示黄、霓虹红等用于悬停、转场、边框、热点。
4. **场景身份**：每个 Collection 拥有独立色彩，进入画廊、场景、详情页后持续影响 UI。
5. **系统状态感**：扫描线、网格、辉光、噪声通过低透明度颜色构建“监控界面”氛围。

---

## 2. 基础色板

基础色定义在 `index.html` 的 CSS 变量中，并通过 Tailwind CDN 配置映射为 `background`、`surface`、`foreground`、`muted`、`border`。

| Token | RGB | HEX 近似值 | 用途 | 语言描述 |
|---|---:|---|---|---|
| `--background` | `5 5 5` | `#050505` | 页面底色、暗角、全局背景 | 深黑、沉浸、无边界 |
| `--surface` | `24 24 27` | `#18181B` | 卡片、面板、抽屉、遮罩层 | 工业面板、金属暗面 |
| `--foreground` | `255 255 255` | `#FFFFFF` | 主标题、主要文本、反选色 | 高对比、强信息 |
| `--muted` | `156 163 175` | `#9CA3AF` | 辅助文字、状态信息 | 冷灰、低权重数据 |
| `--border` | `255 255 255` | `#FFFFFF` | 边框基准色，通常搭配透明度 | 精密线框、界面骨架 |

### 2.1 基础用色原则

- 页面大面积使用 `background`，避免纯亮色铺满。
- `surface` 用于浮层和卡片，透明度通常控制在 `5% - 90%`。
- `foreground` 用于最高优先级信息，但常配合 `/30`、`/50`、`/90` 调整层级。
- `muted` 是默认辅助文本色，适合坐标、状态、说明、元信息。
- `border` 不直接满强度使用，常见为 `border-border/10`、`border-border/20`。

## 4. 首页颜色语言

首页主视觉由 `#CCFF00` 统领，当前在 `HeroSection.tsx` 内作为 `HERO_CONFIG.accentColor` 和多处 HUD 装饰色使用。

### 4.1 首页主色

| 色彩 | 用途 |
|---|---|
| `#CCFF00` | Slogan 背景、HUD 点、扫描线、辉光、噪声高亮 |
| `#FFFFFF` | 大标题、时钟、主文字、点阵默认字符 |
| `#050505` / `black` | 背景遮罩、对比底色 |
| `rgba(204,255,0,0.1 - 0.6)` | 光带、阴影、扫描线、动效辉光 |

### 4.2 语言总结

首页像一个“作品集系统入口”：

- 酸性绿表示系统已激活、访问已授权。
- 黑底让图像和文字有强烈舞台感。
- 白色和灰色承担档案信息的可读性。
- 绿色光效必须克制，以线、点、扫描为主，不做大面积背景。

## 5. Collection / Gallery 动态色语言

作品集列表中的每个 Collection 都有自己的 `color`。这些颜色会驱动：

- 卡片顶部和底部装饰线。
- 卡片 ID、subtitle、hover 图形。
- 画廊卡片边框与文字高亮。
- 页面背景环境光。
- 全屏转场色。

### 5.1 Collection 色彩职责

| 层级 | 颜色行为 |
|---|---|
| 默认状态 | 使用黑、灰、白维持统一界面 |
| Hover 状态 | Collection 色进入边框、文字、图形、背景辉光 |
| Select 状态 | Collection 色成为转场主色 |
| Gallery 状态 | Collection 色成为整个分区的身份色 |

### 5.2 语言总结

Collection 色不是“卡片颜色”，而是“区域身份色”。

用户悬停某个作品时，页面的环境光也会跟随变化，形成“选择哪一组，系统就切换到哪一种频谱”的体验。

## 6. Scene Viewer 颜色语言

场景页中，颜色来自当前 `scene.color`，由 `getScenesForCollection` 从 Collection 色或场景自定义色解析。

### 6.1 场景页颜色用途

| 元素 | 色彩规则 |
|---|---|
| 场景标题阴影 | 使用 `scene.color` 的透明阴影 |
| 状态点 | 使用 `scene.color` 实色 |
| 热点激活态 | 使用 `scene.color` 填充与发光 |
| 瞄准器边角 | 使用 `scene.color` 边框 |
| 详情卡边框 | 使用 `accentColor` |
| 3D 模型辅助光 | 使用 `accentColor` |
| 全屏详情转场 | 使用 `accentColor` |

### 6.2 语言总结

场景页颜色承担“扫描目标锁定”的含义。

当热点激活后，背景图降亮、变灰、轻微模糊，强调色集中到热点、瞄准器、详情卡上，让用户注意力进入分析模式。

## 7. GridOverlay 环境光语言

`GridOverlay` 是全站持续存在的背景视觉系统，负责将当前状态转译为环境光、扫描线与低透明纹理。

### 7.1 颜色逻辑

| 状态 | Primary | Secondary | 视觉含义 |
|---|---|---|---|
| 默认 | `#00F0FF` | `#FF003C` | 冷蓝扫描 + 红色警戒 |
| Collection Hover | 当前 Collection 色 | 当前 Collection 色 | 频谱跟随 |
| Scene Selected | 当前 Scene 色 | 当前 Scene 色 | 场景身份延续 |
| Hotspot Focus | `#FFFFFF` | `#00F0FF` | 白色锁定 + 蓝色扫描 |

### 7.2 语言总结

`GridOverlay` 不负责信息阅读，而负责“氛围同步”。

它通过低透明度径向渐变、扫描线、地形纹理和噪声，把当前页面状态转译成环境光。

## 8. 预加载颜色语言

`Preloader` 使用一套相对独立的红白黑色板。

| 色彩 | 用途 | 语言描述 |
|---|---|---|
| `#000000` | 预加载背景 | 强制幕布、开场黑场 |
| `#F2EDE6` | 标题、文字、纸板 | 温暖白、档案纸张、实体质感 |
| `#D71913` | 板块色 | 红色警示、品牌冲突点 |
| `#FF1D12` | 进度点 | 加载状态、启动信号 |

### 8.1 语言总结

`Preloader` 与主界面的赛博冷光不同，更像“档案封面 / 作品集开场卡”。

它用暖白和红色建立视觉冲击，再过渡到主界面的黑底酸绿系统。

## 9. 交互颜色规则

### 9.1 Hover

- Hover 不改变大面积背景。
- Hover 优先改变：
  - 边框
  - 小型几何符号
  - 标签文字
  - 底部访问按钮
  - 环境光

### 9.2 Active / Selected

- 选中后颜色应集中在目标附近。
- 场景图像进入低亮/灰度状态，强调色用于：
  - 热点
  - 瞄准器
  - 详情卡边框
  - 3D 模型 HUD

### 9.3 Transition

- 页面切换时使用当前 Collection 或 Scene 的强调色作为中间转场层。
- 黑色和深灰作为转场底层，确保强调色不刺眼。

### 9.4 Disabled / Muted

- 辅助信息统一使用 `muted` 或 `foreground/30 - foreground/50`。
- 不建议为弱信息引入新的彩色。

## 10. 推荐色彩规范

### 10.1 使用比例

| 类型 | 推荐比例 |
|---|---:|
| 黑 / 深灰背景 | 70% - 85% |
| 白 / 灰文本与线框 | 10% - 20% |
| 强调色 | 3% - 8% |
| 高强度辉光 | 1% - 3% |

### 10.2 禁止事项

- 不建议大面积铺满 `#CCFF00`、`#00F0FF`、`#FF003C`。
- 不建议同时在同一区块使用 3 个以上高饱和强调色。
- 不建议让正文使用强调色，强调色应优先服务交互和状态。
- 不建议新增相近但不统一的绿色、蓝色、红色，应优先复用 `ACCENT_COLORS`。

### 10.3 新模块接入规则

新增模块时建议遵守：

1. 背景使用 `bg-background` 或 `bg-surface`。
2. 正文使用 `text-muted`，标题使用 `text-foreground`。
3. 边框使用 `border-border/10` 或 `border-border/20`。
4. 交互强调色从外部传入 `accentColor` 或 `--accent-color`。
5. 不在组件内硬编码新色，除非它是独立视觉系统，例如 `Preloader`。

## 11. 可沉淀为设计 Token 的颜色

后续如果要规范化，可将当前颜色拆成以下 Token：

```ts

  base: {
    background: '#050505',
    surface: '#18181B',
    foreground: '#FFFFFF',
    muted: '#9CA3AF',
    border: '#FFFFFF',
  },
  accent: {
    acidGreen: '#CCFF00',
    electricBlue: '#00F0FF',
    neonRed: '#FF003C',
    warningYellow: '#FCEE0A',

    black: '#000000',

};
```
