# 3D模型查看器 - 功能实现文档

## 功能概述

在详情卡片（DetailCard）的全屏展开视图中，所有热点详情卡片均内嵌 3D 模型查看器，并提供放大为独立全屏 3D 查看器的能力。

**核心能力**：
1. 所有热点详情统一使用 **3D 模型查看器**，不再区分图片/3D 双模式
2. 展开态始终内嵌 ModelViewer + 放大按钮
3. 独立全屏 3D 模型查看器弹窗
4. 真实 `.glb` 模型加载（通过 `modelUrl`），无模型时显示占位几何体
5. 统一的赛博朋克 UI 风格与交互语言

## 架构设计

### 组件层级

```
DetailCard.tsx (热点详情卡片)
├── 浮动卡片态 (isExpanded=false)
│   ├── Header: "细节_区块_H3"
│   ├── Image Thumbnail → 点击展开
│   └── Content: 描述文字
│
├── 全屏展开态 (isExpanded=true) → Portal 到 document.body
│   ├── 左侧: 3D 视口 (ModelViewer，始终渲染)
│   │   ├── 放大按钮 → 打开 FullscreenModelViewer
│   │   └── HUD: 旋转角度、状态信息
│   └── 右侧: 信息面板 (标题、描述、关闭按钮)
│
└── 全屏 3D 查看器 (isFullscreenViewer=true) → Portal 到 document.body
    ├── 3D 视口 (ModelViewer, hudPadding=48px)
    └── 信息面板 (标题、描述、操作指南、关闭按钮)
```

### 数据流

```
types.ts (Hotspot.modelUrl?)
  ↓
constants.ts (getScenesForCollection → hotspot.modelUrl)
  ↓
SceneViewer.tsx (activeHotspot → DetailCard)
  ↓
DetailCard.tsx (始终渲染 ModelViewer)
  ├── modelUrl 有值 → ModelViewer 加载真实 .glb 模型
  └── modelUrl 无值 → ModelViewer 渲染 PlaceholderModel 占位几何体
      ↓
  点击放大 → <FullscreenModelViewer modelUrl={hotspot.modelUrl} />
```

## 实现细节

### 1. 数据层 — `types.ts`

Hotspot 接口包含可选 `modelUrl` 字段：

```typescript
export interface Hotspot {
  id: string;
  x: number;
  y: number;
  title: string;
  description: string;
  detailImage: string;
  modelUrl?: string; // .glb / .gltf 3D 模型路径，未配置时显示占位几何体
}
```

### 2. 3D 模型查看器 — `ModelViewer.tsx`

**核心组件**，被展开态和全屏弹窗复用。

```typescript
interface ModelViewerProps {
  accentColor: string;   // 主题色（影响几何体、灯光、HUD）
  modelUrl?: string;     // .glb 模型路径，有值时加载真实模型，无值时显示占位几何体
  className?: string;     // 容器额外样式
  showHud?: boolean;      // 是否显示 HUD 叠加层
  hudPadding?: string;    // HUD 内边距（避免与外部 UI 重叠）
}
```

**场景构成**：
- 真实模型：`useGLTF(url)` 加载 `.glb` 文件，自动居中与缩放
- 占位几何体：`dodecahedronGeometry`（十二面体）+ `MeshDistortMaterial`（扭曲材质）
- 线框外壳：同尺寸 wireframe，增加科技感
- 环绕光环：两个水平圆环，模拟能量场
- 网格地面：`gridHelper`，提供空间参照
- 光照：`ambientLight` + `directionalLight` + `pointLight`（accent color）
- 环境贴图：`<Environment preset="city" />` 提供反射

**模型加载策略**：
- 使用 `<Suspense>` 包裹模型内容，加载中显示旋转动画 + "加载模型..." 文字
- `LoadedModel` 组件：通过 `useGLTF` 加载模型，自动计算包围盒居中缩放
- `PlaceholderModel` 组件：无 `modelUrl` 时的默认占位几何体

**交互控制**：
- `OrbitControls`：左键旋转、滚轮缩放
- `enablePan={false}`：禁用平移
- `minDistance=3 / maxDistance=12`：缩放范围限制
- `maxPolarAngle=Math.PI/1.5`：防止翻转到底部

**HUD 叠加层**（`showHud=true` 时显示）：
- 左上：`3D_VIEWPORT` 标签 + 状态指示灯
- 右上：实时旋转角度 `ROT X:° Y:°`
- 左下：`ORBIT_CTRL` 操作提示
- 右下：`LIVE` 状态

### 3. 全屏 3D 模型弹窗 — `FullscreenModelViewer.tsx`

**独立全屏模态框**，通过 `createPortal` 渲染到 `document.body`。

```typescript
interface FullscreenModelViewerProps {
  title: string;
  description: string;
  accentColor: string;
  modelUrl?: string;
  onClose: () => void;
  isMobile?: boolean;
}
```

**布局**：
- 桌面端：`flex-row`，左侧 3D 视口（flex-1）+ 右侧信息面板（340px）
- 移动端：`flex-col`，上方 3D 视口 + 下方可滚动信息面板（`max-h-[55vh] overflow-y-auto`）
- 普通移动端 3D 视口使用 `dvh` 最小高度，避免移动浏览器动态地址栏或 flex 分配导致 Canvas 高度塌陷

**视觉效果**：
- 转场动画：白色闪烁 + accent color 擦除（`fs-reveal-swipe`，从右向左）
- 信息面板：`slide-in-from-right` 700ms 延迟进入
- 关闭按钮：500ms 延迟淡入

### 4. 堆叠上下文修复 — `createPortal`

**问题**：SceneViewer 使用 `fixed inset-0 z-20` 创建了堆叠上下文，其子组件中的模态框（`z-[100]`、`z-[200]`）无法突破父级 z-index。

**解决方案**：使用 React `createPortal` 将模态框渲染到 `document.body`，脱离 SceneViewer 的 z-20 堆叠上下文。

### 5. 按钮交互优化

全宽操作按钮统一采用三态反馈：

```typescript
bg-foreground text-background shadow-lg
hover:bg-[--accent-color] hover:shadow-xl hover:scale-[1.01]
active:scale-[0.97] active:brightness-90
transition-all duration-300
```

### 6. HUD 间距控制

ModelViewer 新增 `hudPadding` prop，通过 CSS `padding` 控制 HUD 元素与视口边缘的距离，避免与外部按钮（关闭、放大）重叠：

```typescript
<ModelViewer hudPadding="48px" />
```

### 7. 移动端 Canvas 尺寸修复

移动端展开态与独立全屏态都需要由外层 3D 视口提供稳定高度，`ModelViewer` 自身只继承父级尺寸并保持 `min-h-0 overflow-hidden`。这样可以避免移动浏览器动态视口、Portal 动画或 flex 子项分配导致 R3F Canvas 初始化时高度为 0。

| 场景 | 尺寸策略 |
|------|----------|
| DetailCard 展开态 | 普通移动端设置 `min-h-[42dvh]`，桌面端设置 `md:min-h-[520px]` |
| FullscreenModelViewer | 普通移动端设置 `min-h-[42dvh] sm:min-h-[50dvh]` |
| ModelViewer 根容器 | `w-full h-full min-h-0 overflow-hidden` |

## 侧边栏动画（GSAP 单 Timeline + addPause）

### 概述

SceneViewer 右侧 Sidebar 的展开/收起动画原为 CSS `transition-all` + `transition-opacity`（双层叠加 + 延迟错峰），存在以下问题：

- CSS transition 不可中断，hover in/out 快速切换时容易跳变
- 列表项全部同时出现，缺乏层次
- 进入与退出节奏相同，反馈不够"果断"

参考 [GSAP 官方 Demo: Interruptible Single Timeline Enter/Exit](https://demos.gsap.com/demo/interruptible-single-timeline-enterexit/)，重构为「单 timeline + `addPause()` + 双段 enter/exit」模式。

### 实现要点

| 段 | 动画对象 | 桌面端 | 移动端 |
|----|----------|--------|--------|
| **enter** | aside 容器 | `width: 60 → 320` | `xPercent: 100 → 0` |
| | 折叠条 60px | `opacity: 1 → 0` | 隐藏 |
| | 内容区 | `opacity: 0 → 1`, `x: 12 → 0`，延迟 0.18s | 同 |
| | 热点列表 | stagger 0.045s 错落进入 | 同 |
| `addPause()` | — | 等待用户操作 | 同 |
| **exit** | 热点列表 | stagger 0.018s 错落退出 | 同 |
| | 内容区 | `opacity: 1 → 0`, `x: 0 → 12` | 同 |
| | aside 容器 | `width: 320 → 60` | `xPercent: 0 → 100` |
| | 折叠条 60px | `opacity: 0 → 1` | 显示 |

### 缓动对照

| 段 | ease | 设计意图 |
|----|------|----------|
| enter | `back.out(1.2)` | 弹性回弹，体现"到位感" |
| enter（内容/列表） | `power2.out` | 标准减速，配合 stagger 错落 |
| exit | `power3.in` | 急速撤离，撤销操作响应干脆 |
| exit（折叠条淡入） | `power2.in` | 与容器退场同步 |

### 可中断性原理

```ts
// isMenuOpen: true →  tl.play()   // 从 0 位置正向播到 addPause 处停下
// isMenuOpen: false → tl.reverse() // 从当前时间反向播回 0
// 反复 hover 时，tl 总是在当前位置继续 —— 天然中断
```

`addPause()`（默认 `'>'` 位置 = timeline 末尾）将 enter 与 exit 两段在时间上分隔。用户停留时停留在 enter 段末尾；改变主意时调用 `reverse()`，从 enter 段末尾反向走回起点。

### 关键技术细节

1. **常量集中**：所有动画参数集中在文件顶部 `SIDEBAR_ANIM` 对象，便于调优。
2. **refs 而非 querySelector**：`asideRef` / `collapsedBarRef` / `contentRef` / `listItemRefs` 均为 React ref，避免 GSAP 作用域污染。
3. **listItemRefs 数组**：通过 `ref={(el) => { listItemRefs.current[idx] = el; }}` 收集 9 个列表按钮，传给 `gsap.to(items, { stagger: ... })`。
4. **pointer-events 同步**：useEffect 中根据 `isMenuOpen` 同步设置 `contentRef.style.pointerEvents`，避免折叠态下不可见的内容区子元素拦截点击。
5. **prefers-reduced-motion 分支**：检测到系统级"减少动画"偏好时直接 `gsap.set` 同步样式，不创建 timeline。
6. **重建策略**：`useGSAP` 依赖 `[prefersReducedMotion, isMobile, scene.id]`，桌面/移动模式切换或场景切换时会重建 timeline。
