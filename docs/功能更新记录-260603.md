### 13. 首页顶栏 GSAP 动效重构 (`HomeSelection.tsx`)

**核心改进**:
- **玻璃态底层**: `bg-background/30 dark:bg-background/10` + `backdrop-blur-xl dark:backdrop-blur-2xl` + `backdrop-saturate-150`，移除关键帧 CSS 动画
- **流动渐变层**: 底层 `#CCFF00` 绿光晕（`blur 28px`）+ 底部流光线（带绿色辉光 `box-shadow`），由 GSAP 驱动 `backgroundPosition` 14s/12s 循环
- **3D 倾斜 + 视差**: 标题组跟随鼠标做 `rotateX/rotateY`（`perspective 800`，`max 12°`），标签/标题按系数 3/6 做反向 xy 视差
- **弹性回弹**: 鼠标离开时用 `elastic.out(1, 0.5)` 0.8s 归零，避免硬切
- **悬停加速**: hover 时 `continuousTl.timeScale → 2.5`，光晕 `opacity 0.7 → 0.85`

**关键常量**（文件顶部）:
```typescript
const TILT_MAX_DEG = 12;          // 顶栏标题组最大倾斜角度
const TILT_PERSPECTIVE = 800;     // 3D 透视距离
const PARALLAX_FACTOR_LABEL = 3;  // "场景目录"标签视差系数
const PARALLAX_FACTOR_TITLE = 6;  // 大标题视差系数
```

**技术要点**:
- 入场动画用一条 timeline 串联：光晕 → 流光线 → 脉冲点 → 标签 → 标题 → 元数据
- 持续动画用独立 timeline 循环，**悬停加速 2.5x** 通过 `gsap.to(continuousTl, { timeScale })` 实现
- 所有鼠标驱动的 `gsap.to` 均加 `overwrite: 'auto'` 避免相互覆盖

### 14. 斜切标语 3D 交互 (`HomeSelection.tsx`)

**核心改进**:
- **斜切 + 3D 倾斜**: 容器 `skewX -12` + `transformOrigin bottom right` + `transformPerspective 600`，hover/move 时叠加 `rotateX/Y`（`max 15°`）
- **文字视差**: sloganText 用 `skewX 12` 反向补偿 + 系数 4 反向 xy 视差，避免文字被斜切"压扁"
- **高光扫过**: shine 层 `linear-gradient 105deg`，hover 时 `xPercent -100 → 100` 一次
- **字间距扩展**: hover/move 时 `letterSpacing: '0.3em' → '0.35em'`，配 `textShadow` 暖色阴影
- **动态阴影**: 阴影偏移跟随倾斜方向 (`shadowX/Y = ±clamped * 8`)，多层叠加形成"光从侧边打来"的效果
- **弹性归零**: 离开用 `elastic.out(1, 0.5)` 0.8s 归零

**关键常量**:
```typescript
const SLOGAN_TILT_MAX_DEG = 15;        // 斜切标签最大倾斜角度
const SLOGAN_TILT_PERSPECTIVE = 600;   // 斜切标签 3D 透视距离
const SLOGAN_PARALLAX_FACTOR = 4;      // 斜切标签文字视差系数
const SLOGAN_SHADOW_OFFSET = 8;        // 斜切标签动态阴影偏移量
```

**踩坑点**:
- 容器 `skewX` 与文字 `skewX` 必须**反号**才能让文字保持水平
- `transformOrigin` 设为 `bottom right` 让倾斜看起来像"右上翘起"
- 鼠标移动时必须在动画里**重设 skewX**（`gsap.set` 的 skewX 会被 to 覆盖），否则松手会"突然变直"

### 15. SceneViewer 侧边栏 GSAP 重构 (`SceneViewer.tsx`)

**核心改进**:
- **可中断 enter/exit**: 单 timeline + `addPause()` 切分 enter/exit，配合 `play()` / `reverse()` 双向控制
- **桌面端**: aside 宽度 60px↔320px + 列表 stagger（`back.out(1.2)` 进入 / `power3.in` 退出）
- **移动端**: 抽屉 `xPercent 100↔0` + 列表 stagger
- **prefers-reduced-motion**: 检测到时直接 `gsap.set` 同步样式，不创建 timeline
- **pointer-events 同步**: useEffect 中根据 `isMenuOpen` 同步设置，避免折叠态下不可见内容区子元素拦截点击

**关键实现**:
```typescript
const tl = gsap.timeline({ paused: true });
// === ENTER 段 ===
tl.to(aside, { width: 320, duration: 0.55, ease: 'back.out(1.2)' }, 0)
  .to(collapsed, { opacity: 0, duration: 0.22 }, 0)
  .to(content, { opacity: 1, x: 0, duration: 0.3, delay: 0.18 }, 0.18)
  .to(items, { opacity: 1, x: 0, duration: 0.32, stagger: 0.045 }, 0.23)
  .addPause();

// === EXIT 段（独立 ease）===
tl.to(items, { opacity: 0, x: 6, duration: 0.18, stagger: 0.018, ease: 'power3.in' }, '>-0.05')
  .to(content, { opacity: 0, x: 12, duration: 0.2, ease: 'power3.in' }, '<0.05')
  .to(aside, { width: 60, duration: 0.3, ease: 'power3.in' }, '<0.04')
  .to(collapsed, { opacity: 1, duration: 0.22, ease: 'power2.in' }, `-=${0.3 * 0.6}`);

// === 状态切换 ===
if (isMenuOpen) tl.play();
else tl.reverse();
```

**所有动画参数集中**在文件顶部 `SIDEBAR_ANIM` 对象，便于调优。

### 17. useGSAP 多分支初始化交叉污染修复

**踩坑场景**: SceneViewer 侧边栏同时支持桌面（hover 展开 60px↔320px）和移动（点击 FAB 抽屉 xPercent 100↔0）两套动画。

**症状**: 桌面端（`isMobile=false`）时，aside 被设两次：先 `width: 60` 后 `xPercent: 100, width: '100%'`。第二次覆盖第一次，aside 整体被推出视口右侧，**完全不可见**。

**根因**: `useGSAP` 内的所有 `gsap.set` / `gsap.timeline` 都是对真实 DOM 的副作用，与 `isMobile` 条件无关。把"桌面端和移动端"写成两段顺序代码，会让两段都执行。

**正确实现**:
```typescript
useGSAP(() => {
  if (prefersReducedMotion) {
    gsap.set(aside, { width: collapsedW, xPercent: 0 });
    return;
  }

  desktopTlRef.current = null;  // 切换时显式置空
  mobileTlRef.current = null;

  if (isMobile) {
    gsap.set(aside, { width: '100%', xPercent: 100 });
    // ... 构建 mobileTl ...
    mobileTlRef.current = mobileTl;
    return;                     // 早返
  }

  gsap.set(aside, { width: collapsedW, xPercent: 0 });
  // ... 构建 desktopTl ...
  desktopTlRef.current = desktopTl;
}, [prefersReducedMotion, isMobile, scene.id]);
```

**三条防线**:
1. **`if/else + 早返` 隔离分支**：桌面端不会跑到移动端 `gsap.set`
2. **互斥 ref 显式置空**：切换 `isMobile` 重建时，旧 ref 不残留
3. **依赖项含 `isMobile`**：模式切换自动重建 timeline

### 8. GSAP 鼠标驱动 transform 的"三件套"配置

所有"鼠标移动驱动 transform"的动画，都必须同时设三个属性：

```typescript
gsap.to(el, {
  rotateX: ..., rotateY: ...,
  transformPerspective: 800,        // 透视距离
  transformOrigin: '50% 50%',       // 旋转原点
  duration: 0.4, ease: 'power2.out',
  overwrite: 'auto',                // 关键：避免 hover/leave 互相覆盖
});
```

**`overwrite: 'auto'`** 是关键：没有它，快速 hover/leave 会让多个 `gsap.to` 互相打架，出现"跳变"。

### 9. backdrop-blur 与 3D 变换的冲突

**问题**: `backdrop-blur-*` 创建独立堆叠上下文，**会强制 `transform-style: flat`**，使子元素的 `preserve-3d` 失效。

**本项目场景**: HomeSelection 顶栏用 `backdrop-blur-xl` 包标题组 → 标题组内的 `rotateX/rotateY` 会因为父级 `transform-style: flat` 而**看起来像 2D 倾斜**，没有透视感。

**解决**:
- **方案 A（推荐）**: 把 `backdrop-blur` 应用在**最外层容器**，让需要 3D 变换的子元素不再被 `transform-style: flat` 约束
- **方案 B（兜底）**: 放弃 `preserve-3d`，用 2D 视差 + `transformPerspective` 模拟深度（`gsap` 的 `transformPerspective` 走的是独立渲染路径，不受 `transform-style` 影响）

本项目使用 **方案 B**：所有倾斜/视差都由 GSAP 驱动，`transformPerspective` 通过 `gsap.set` / `gsap.to` 写入内联 style，**绕开 CSS 堆叠上下文的限制**。

### 10. 斜切 + 3D 倾斜的"双 skew"技巧

容器 `skewX(-12)` 会让内容**整体跟着倾斜**，看起来"歪"。解决：

```typescript
// 容器倾斜
gsap.set(sloganRef.current, {
  skewX: -12,
  transformOrigin: 'bottom right',
  transformPerspective: 600,
});

// 文字反向补偿
gsap.set(sloganTextRef.current, { skewX: 12 });

// 鼠标移动时，**必须**重设容器和文字的 skewX
gsap.to(sloganRef.current, { skewX: -12, rotateX: ..., rotateY: ..., ... });
gsap.to(sloganTextRef.current, { skewX: 12, x: ..., y: ..., ... });
```

**踩坑**: 松开鼠标时 `gsap.to` 不会"保留" `skewX`（如果只设 `rotateX/rotateY: 0`），文字会突然变直。**必须在每次动画里显式带 skewX**。

### 11. 弹性回弹的"过冲"参数

`elastic.out(1, 0.5)` 的两个参数：
- 第一个 `1`：振幅（amplitude），越大过冲越多
- 第二个 `0.5`：周期（period），越小回弹越快

本项目悬停归零统一用 `elastic.out(1, 0.5) duration: 0.8`——既不会"软绵绵"也不会"弹太远"。

### 12. 悬停加速 timeline.timeScale

用 `gsap.to(continuousTl, { timeScale: 2.5, duration: 0.4 })` 让循环动画**平滑加速**，而不是直接 `timeScale(2.5)` 突变。duration 越小加速越突兀，越大越柔和。

### 13. 单 Timeline + addPause() 的"可中断"原理

参考 [GSAP Demo: Interruptible Single Timeline Enter/Exit](https://demos.gsap.com/demo/interruptible-single-timeline-enterexit/)

```typescript
const tl = gsap.timeline({ paused: true });
// enter 段
tl.to(...).to(...).addPause();  // 停在 enter 末尾
// exit 段（独立 ease）
tl.to(...).to(...);

// isOpen=true  → tl.play()     // 从 0 正向播到 addPause 停下
// isOpen=false → tl.reverse()  // 从 addPause 反向播回 0
// 反复 hover 时，tl 总是在 tl.time() 当前位置继续——天然中断
```

**注意**: 单 timeline 的 `reverse()` 会自动反转 enter 段的 ease（`back.out` → `back.in`），这与"显式 exit 段"的体验略不同。**推荐方案**：把 exit 段独立写出来，用完全不同的 `power3.in`，让"撤销"感觉更干脆。

### 2026-06-02 代码优化技术要点

### 15. Vite `manualChunks` 代码分割

**问题**: Three.js + R3F + Drei 打包为单个 1.2MB chunk，首屏加载慢。

**解决方案**:
```typescript
// vite.config.ts
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-three': ['three', '@react-three/fiber', '@react-three/drei'],
        'vendor-gsap': ['gsap', '@gsap/react'],
      }
    }
  }
}
```

**效果**: Three.js 和 GSAP 各自独立 chunk，支持浏览器并行加载 + 长期缓存。

### 16. 内联 `@keyframes` 统一管理

**问题**: 4 个组件各自用 `<style>` 内联 `@keyframes`，重复定义且难以维护。

**解决**: 全部迁移到 `index.html` 的 `<style>` 块，组件中删除 `<style>` 标签。动画类名不变，无需修改 className。

### 17. `useCallback` 包装事件处理器

**问题**: App.tsx 中 `handleSelectCollection` 等函数每次渲染都重新创建，导致子组件无效重渲染。

**解决**: 用 `useCallback` 包装，依赖项为 `navigate`（稳定的函数引用）和必要的状态。

### 2026-06-03 技术要点

### 18. GSAP 轮播无缝循环

首图重复追加到末尾（`[...images, images[0]]`），timeline 在最后一张停留后 `gsap.set` 归零 `yPercent: 0`，视觉上无跳变。

### 19. 点阵噪声 ASCII 字符网格

**实现**: `createDotNoiseCells` 函数按 `DOT_GRID_SIZE` 网格划分区域，随机选取 `activeRate` 比例的格子填充字符。15% 概率使用强调色，其余白色。`setInterval` 每 180ms 重新生成，制造"闪烁"效果。

**性能**: 896×411 视口下约 44×20 = 880 格，2% 活跃率仅 17 个 DOM 节点，5% 也仅 44 个。

### 20. playTextScramble 文本乱码解码

**原理**: `requestAnimationFrame` 逐帧计算 `progress = (now - start) / duration`，每个字符按 `index / length + random * 0.28` 阈值决定是否揭示。`random` 每帧不同，产生"逐字解密"的随机感。

**关键**: 保留空格字符不扰乱，最终 `progress >= 1` 时强制设为 `sourceText`。

### 21. LERP 鼠标跟随

`requestAnimationFrame` + 线性插值 `current += (target - current) * 0.14`。0.14 系数越小跟随越"粘滞"，越大越"灵敏"。配合视口边界限制和速度旋转，形成有物理感的跟随效果。
