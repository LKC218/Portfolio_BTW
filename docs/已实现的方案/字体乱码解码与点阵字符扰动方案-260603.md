# 字体乱码解码与点阵字符扰动方案-260603

## 1. 方案定位

本文档记录首页已实现的字体动态效果方案，用于后续在其他页面或组件中复用同类交互。

当前实现位置：

- `src/components/pages/HomeSelection.tsx`

已实现效果包括：

1. HUD 小字 hover 乱码解码。
2. CTA 标语 hover 乱码解码，并保留原有 3D 倾斜和流光效果。
3. 首页白点点阵默认低密度 ASCII 字符扰动。
4. 点阵层 hover 时整体增强扰动密度。

不包含：

- 时间数字乱码效果。该效果已按需求移除，`RollingClock` 只保留原有滚动翻页时钟。

---

## 2. 交互原则

### 2.1 文字乱码解码

适用对象：

- 按钮文案
- HUD 状态文字
- 系统提示文字
- 小字号终端风格文字

不建议用于：

- 大段正文
- 实时时钟数字
- 需要稳定阅读的主要标题

核心原则：

- 只在 hover 或短时间反馈中触发。
- 动效时长控制在 `300ms ~ 600ms`。
- 结束后必须恢复最终文本。
- 遵守 `prefers-reduced-motion`。

### 2.2 点阵字符扰动

适用对象：

- 已有规则点阵背景
- 科技感 HUD 背景
- 首页 Hero 图片上方的轻量信息纹理

核心原则：

- 字符必须匹配点阵网格位置。
- 字符透明度必须低，不能遮挡背景图片。
- 默认状态低密度扰动。
- hover 只做整体增强，不跟随鼠标生成局部块。

## 3. 文字 scramble 解码实现

### 3.1 常量配置

```tsx
const SCRAMBLE_CHARS = '/\\|_+-=<>:;.*#01X';
const SCRAMBLE_DURATION = 520;
```

字符池说明：

- `/`、`\\`、`|`：终端扫描感。
- `_`、`-`、`+`、`=`：系统界面感。
- `<`、`>`、`:`、`;`：命令行结构感。
- `0`、`1`、`X`：数字噪声感。

### 3.2 核心函数

```tsx
const playTextScramble = (
  element: HTMLElement | null,
  finalText: string,
  duration = SCRAMBLE_DURATION
) => {
  if (!element || window.matchMedia('(prefers-reduced-motion: reduce)').matches) return;

  const start = performance.now();
  const sourceText = finalText;

  const tick = (now: number) => {
    const progress = Math.min((now - start) / duration, 1);
    const nextText = sourceText
      .split('')
      .map((char, index) => {
        if (char === ' ') return ' ';
        const revealThreshold = index / sourceText.length;
        if (progress > revealThreshold + Math.random() * 0.28) return char;
        return SCRAMBLE_CHARS[Math.floor(Math.random() * SCRAMBLE_CHARS.length)];
      })
      .join('');

    element.textContent = progress >= 1 ? sourceText : nextText;

    if (progress < 1) {
      requestAnimationFrame(tick);
    }
  };

  requestAnimationFrame(tick);
};
```

### 3.3 调用方式

先定义目标元素引用：

```tsx
const statusTotalRef = useRef<HTMLDivElement>(null);
```

封装 hover 调用：

```tsx
const handleHudScramble = useCallback((element: HTMLElement | null, text: string) => {
  playTextScramble(element, text, 420);
}, []);
```

绑定到元素：

```tsx
<div
  ref={statusTotalRef}
  onMouseEnter={() => handleHudScramble(statusTotalRef.current, `TOTAL_SECTORS: ${collections.length}`)}
>
  TOTAL_SECTORS: {collections.length}
</div>
```

### 3.4 当前已接入文本

首页当前已接入：

- `TOTAL_SECTORS`
- `SCROLL_MODE`
- `//.. ACCESS_GRANTED`
- `SYSTEM_ROOT` 底部路径文本
- `LIVE // Personal Work`
- `< 作品集 // LKC218 >`

## 4. CTA 标语组合动效

CTA 标语同时包含三类反馈：

1. 文本 scramble 解码。
2. 容器 3D 倾斜。
3. 高光流光扫过。

### 4.1 hover 进入时先触发 scramble

```tsx
const handleSloganMouseEnter = useCallback(() => {
  if (sloganTextRef.current) {
    playTextScramble(sloganTextRef.current, `< ${HERO_CONFIG.slogan} >`);
  }

  if (prefersReducedRef.current || !sloganRef.current) return;

  gsap.to(sloganRef.current, {
    scale: 1.05,
    duration: 0.3,
    ease: 'back.out(2)',
    overwrite: true
  });
}, []);
```

### 4.2 mouse leave 时恢复最终文本

```tsx
const handleSloganMouseLeave = useCallback(() => {
  if (sloganTextRef.current) {
    sloganTextRef.current.textContent = `< ${HERO_CONFIG.slogan} >`;
  }

  if (sloganShineRef.current) {
    gsap.killTweensOf(sloganShineRef.current);
    gsap.set(sloganShineRef.current, { xPercent: -100 });
  }
}, []);
```

注意：

- 离开时必须恢复最终文本，避免中途离开导致乱码残留。
- `sloganShineRef` 需要重置到起点，避免高光停在中间。

## 5. 白点点阵 ASCII 扰动实现

### 5.1 当前视觉目标

当前点阵扰动不是鼠标附近的局部矩阵，而是：

- 默认整层存在轻微扰动。
- hover 点阵层时整体增强。
- 字符严格对齐白点网格。
- 低透明度，不遮挡背景图片。

### 5.2 常量配置

```tsx
const DOT_GRID_SIZE = 20;
const DOT_NOISE_REFRESH_MS = 180;
const DOT_DEFAULT_ACTIVE_RATE = 0.02;
const DOT_HOVER_ACTIVE_RATE = 0.05;
const DOT_ASCII_CHARS = ['.', ':', '-', '0', '4', '6', '8', '9', 'A', '/', '\\', '+'];
```

参数说明：

| 参数 | 作用 |
|---|---|
| `DOT_GRID_SIZE` | 必须与背景点阵 `bg-[size:20px_20px]` 一致 |
| `DOT_NOISE_REFRESH_MS` | 字符扰动刷新频率 |
| `DOT_DEFAULT_ACTIVE_RATE` | 默认活跃字符比例 |
| `DOT_HOVER_ACTIVE_RATE` | hover 时活跃字符比例 |
| `DOT_ASCII_CHARS` | 点阵扰动字符池 |

### 5.3 数据结构

```tsx
interface DotNoiseCell {
  id: string;
  x: number;
  y: number;
  char: string;
  opacity: number;
  color: string;
}
```

每个 `DotNoiseCell` 只表示一个当前活跃字符，不表示所有白点。

这样可以避免整屏渲染过多节点。

### 5.4 生成扰动字符

```tsx
const createDotNoiseCells = (
  width: number,
  height: number,
  activeRate: number
): DotNoiseCell[] => {
  const cols = Math.ceil(width / DOT_GRID_SIZE);
  const rows = Math.ceil(height / DOT_GRID_SIZE);
  const total = cols * rows;
  const activeCount = Math.max(1, Math.floor(total * activeRate));
  const used = new Set<number>();

  return Array.from({ length: activeCount }, (_, index) => {
    let cellIndex = Math.floor(Math.random() * total);
    while (used.has(cellIndex)) {
      cellIndex = Math.floor(Math.random() * total);
    }
    used.add(cellIndex);

    const col = cellIndex % cols;
    const row = Math.floor(cellIndex / cols);
    const isAccent = Math.random() < 0.15;

    return {
      id: `${index}-${cellIndex}`,
      x: col * DOT_GRID_SIZE,
      y: row * DOT_GRID_SIZE,
      char: DOT_ASCII_CHARS[Math.floor(Math.random() * DOT_ASCII_CHARS.length)],
      opacity: isAccent ? 0.14 + Math.random() * 0.04 : 0.08 + Math.random() * 0.05,
      color: isAccent ? '#CCFF00' : '#FFFFFF'
    };
  });
};
```

关键点：

- `x = col * DOT_GRID_SIZE`
- `y = row * DOT_GRID_SIZE`
- 坐标严格匹配白点网格。
- 只渲染活跃字符，不渲染全量白点。
- 绿色强调字符比例控制在 `15%`。

### 5.5 状态与刷新

```tsx
const [dotNoiseCells, setDotNoiseCells] = useState<DotNoiseCell[]>([]);
const [isDotNoiseHovering, setIsDotNoiseHovering] = useState(false);
const dotOverlayRef = useRef<HTMLDivElement>(null);
```

刷新逻辑：

```tsx
useEffect(() => {
  if (prefersReducedRef.current || !dotOverlayRef.current) return;

  const updateDotNoise = () => {
    if (!dotOverlayRef.current) return;
    const rect = dotOverlayRef.current.getBoundingClientRect();
    const activeRate = isDotNoiseHovering ? DOT_HOVER_ACTIVE_RATE : DOT_DEFAULT_ACTIVE_RATE;
    setDotNoiseCells(createDotNoiseCells(rect.width, rect.height, activeRate));
  };

  updateDotNoise();
  const intervalId = window.setInterval(updateDotNoise, DOT_NOISE_REFRESH_MS);

  return () => window.clearInterval(intervalId);
}, [isDotNoiseHovering]);
```

### 5.6 点阵层结构

```tsx
<div
  ref={dotOverlayRef}
  className="absolute inset-0 bg-[radial-gradient(rgba(255,255,255,0.15)_1px,transparent_1px)] bg-[size:20px_20px] pointer-events-auto overflow-hidden"
  onMouseEnter={() => setIsDotNoiseHovering(true)}
  onMouseLeave={() => setIsDotNoiseHovering(false)}
>
  <div className="absolute inset-0 font-mono select-none pointer-events-none mix-blend-screen">
    {dotNoiseCells.map((cell) => (
      <span
        key={cell.id}
        className="absolute flex items-center justify-center transition-opacity duration-150"
        style={{
          left: cell.x,
          top: cell.y,
          width: DOT_GRID_SIZE,
          height: DOT_GRID_SIZE,
          fontSize: '9px',
          lineHeight: `${DOT_GRID_SIZE}px`,
          color: cell.color,
          opacity: cell.opacity,
          textShadow: cell.color === '#CCFF00' ? '0 0 4px rgba(204,255,0,0.2)' : 'none'
        }}
      >
        {cell.char}
      </span>
    ))}
  </div>
</div>
```

## 6. 透明度与可读性规范

### 6.1 点阵扰动透明度

推荐范围：

| 类型 | 透明度 |
|---|---:|
| 普通白色字符 | `0.08 ~ 0.13` |
| 强调绿色字符 | `0.14 ~ 0.18` |
| 白点背景 | `rgba(255,255,255,0.15)` |

如果背景图片较暗，可以略微提高字符透明度。

如果背景图片信息量较大，应优先降低 `DOT_HOVER_ACTIVE_RATE`，不要只降低透明度。

### 6.2 活跃比例

推荐范围：

| 状态 | 活跃比例 |
|---|---:|
| 默认 | `0.015 ~ 0.025` |
| hover | `0.035 ~ 0.06` |

不建议超过 `0.08`，否则字符会开始抢背景图。

## 7. 复用步骤

在新页面复用时，按以下步骤迁移：

1. 复制 `SCRAMBLE_CHARS`、`SCRAMBLE_DURATION` 和 `playTextScramble`。
2. 给目标文本元素建立 `useRef`。
3. 在 `onMouseEnter` 中调用 `playTextScramble(ref.current, finalText)`。
4. 若目标元素会在 hover leave 中被 GSAP 或其他动画影响，离开时手动恢复 `textContent`。
5. 如需点阵扰动，复制 `DOT_GRID_SIZE`、`DotNoiseCell`、`createDotNoiseCells` 和刷新 `useEffect`。
6. 确认点阵背景的 `bg-[size:...]` 与 `DOT_GRID_SIZE` 一致。
7. 根据背景复杂度调整透明度和活跃比例。
8. 保留 `prefers-reduced-motion` 判断。

## 8. 常见问题

### 8.1 乱码结束后文本残留错误

原因：

- 鼠标快速移出时动画还没结束。

处理：

- 在 `onMouseLeave` 中显式恢复最终文本。

### 8.2 点阵字符和白点错位

原因：

- `DOT_GRID_SIZE` 与背景 `bg-[size:...]` 不一致。
- 字符单元没有使用相同宽高。

处理：

- 确保 `DOT_GRID_SIZE = 20` 时背景为 `bg-[size:20px_20px]`。
- 字符 `width`、`height`、`lineHeight` 都使用 `DOT_GRID_SIZE`。

### 8.3 字符影响背景图片

处理优先级：

1. 降低 `DOT_HOVER_ACTIVE_RATE`。
2. 降低 `DOT_DEFAULT_ACTIVE_RATE`。
3. 降低 `opacity`。
4. 减少绿色强调字符比例。

### 8.4 性能压力增加

处理：

- 降低活跃比例。
- 增大 `DOT_NOISE_REFRESH_MS`。
- 只在桌面端启用完整扰动。
- 不渲染所有网格点，只渲染活跃字符。
