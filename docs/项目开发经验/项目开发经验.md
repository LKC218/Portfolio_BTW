## 架构设计经验

### 1. 状态管理模式

项目采用**集中式状态管理**，所有核心状态在 `App.tsx` 中统一管理：

```typescript
// 核心状态
const [appState, setAppState] = useState<AppState>(AppState.HOME);
const [selectedCollection, setSelectedCollection] = useState<Collection | null>(null);
const [selectedScene, setSelectedScene] = useState<Scene | null>(null);
const [hoveredColor, setHoveredColor] = useState<string | null>(null);
const [activeHotspot, setActiveHotspot] = useState<Hotspot | null>(null);
```

**优点**: 状态流向清晰，调试方便  
**缺点**: 大型项目可能导致不必要的重渲染

### 2. 路由系统设计

使用**状态机模式**替代传统路由：

```typescript
enum AppState {
  HOME,    // 首页 - 作品集选择
  GALLERY, // 画廊 - 场景列表
  VIEWER   // 查看器 - 场景详情
}
```

**转场动画实现**: 3层叠加的 slide 动画，通过 CSS animation 和 setTimeout 配合。

### 3. 数据架构

```
constants.ts
├── COLLECTIONS[] (18个作品集)
└── SCENES_DB{} (每个作品集6个场景)
    └── getScenesForCollection() (数据转换)
```

**热区位置**: 使用 `Math.random()` 生成随机偏移，增加视觉随机性。


### 2. HomeSelection 组件

**核心功能**: 首页展示与作品集选择

**视觉特性**:
- Hero 区域: 视差滚动效果
- 网格布局: 3列自适应
- Marquee 分隔线: 滚动驱动的动画

### 3. GridOverlay 组件

**核心功能**: 背景网格效果，响应环境色变化

**实现要点**:
- 使用 CSS background 实现网格
- 通过 CSS 变量动态调整颜色
- 热区聚焦时网格产生呼吸效果

### 4. HeroSection 组件拆分

**背景**: Hero 区域（顶栏 + 背景 + 时钟 + 标语 + 点阵噪声 + HUD）原在 `HomeSelection.tsx` 中，随着功能增长到 800+ 行，需要拆分。

**拆分原则**:
- HeroSection 接收最小 props：`collectionsCount`、`isFullscreen`、`onToggleFullscreen`
- HomeSelection 只负责卡片网格 + ScrollVideo 集成 + 分页逻辑
- 拆分后 HomeSelection 从 800+ 行精简到约 262 行

**经验**: 当单个页面组件超过 500 行且包含多个独立功能区域时，应考虑按视觉区域拆分为独立组件。Hero 区域的动画逻辑（轮播、点阵噪声、scramble）自成体系，与卡片网格无交叉依赖，天然适合拆分。

### 5. HotspotPreview 热点悬停预览

**核心功能**: 桌面端热点悬停时显示详情预览卡片，跟随鼠标移动

**实现要点**:
- 仅桌面端渲染（`isMobile` 时不挂载）
- `requestAnimationFrame` + 线性插值 (LERP 0.14) 平滑跟随
- 视口边界限制防止卡片溢出屏幕
- 基于水平速度的旋转效果增加物理感
- `clip-path inset` + `opacity` 实现展示/隐藏过渡

**经验**: 跟随鼠标类组件用 `requestAnimationFrame` + LERP 比 GSAP `quickTo` 更灵活——可以同时计算速度、做视口限制、叠加旋转。LERP 系数 0.14 是反复调参的结果：太小（<0.1）"粘滞"感过重，太大（>0.2）失去平滑感。

### 6. ScrollVideo 滚动驱动视频

**核心功能**: GSAP ScrollTrigger 驱动的视频展示组件

**实现要点**:
- `gsap.context()` 包裹所有 ScrollTrigger 动画，清理时 `ctx.revert()`
- `IntersectionObserver` (threshold 0.1) 控制离屏暂停/入屏播放
- `video.muted = true` + `playsInline` 确保自动播放策略兼容

**经验**: 视频组件必须同时处理两种生命周期——滚动动画（GSAP ScrollTrigger）和播放控制（IntersectionObserver）。两者职责分离：ScrollTrigger 只管视觉效果（缩放/淡入），IntersectionObserver 只管播放/暂停。用 `gsap.context()` 包裹可确保组件卸载时自动清理所有 ScrollTrigger 实例。

## GSAP 动画经验

### 1. 可中断 enter/exit：单 Timeline + `addPause()`

**场景**：SceneViewer 右侧 Sidebar 展开/收起需要支持 hover 反复触发 + 移动端点击反复触发，且任何一次操作都应能即时中断上一次动画。

**错误方案**：
- 两条独立 timeline（enterTl / exitTl）：在动画中途切换时，新 timeline 从 0 开始播，旧 timeline 还在原地 → 视觉打架
- CSS `transition-all` + 多次 className 切换：不可中断，快速 hover 时元素属性跳变无过渡

**正确方案**：单条 timeline + `addPause()` 切分 enter/exit，配合 `play()` / `reverse()` 双向控制。

```ts
const tl = gsap.timeline({ paused: true });
// === ENTER 段 ===
tl.to(aside, { width: 320, duration: 0.55, ease: 'back.out(1.2)' }, 0)
  .to(collapsed, { opacity: 0, duration: 0.22 }, 0)
  .to(content, { opacity: 1, x: 0, duration: 0.3, delay: 0.18 }, 0.18)
  .to(items, { opacity: 1, x: 0, duration: 0.32, stagger: 0.045 }, 0.23)
  .addPause();                                    // 关键：等待用户操作

// === EXIT 段（独立 ease）===
tl.to(items, { opacity: 0, x: 6, duration: 0.18, stagger: 0.018, ease: 'power3.in' }, '>-0.05')
  .to(content, { opacity: 0, x: 12, duration: 0.2, ease: 'power3.in' }, '<0.05')
  .to(aside, { width: 60, duration: 0.3, ease: 'power3.in' }, '<0.04')
  .to(collapsed, { opacity: 1, duration: 0.22, ease: 'power2.in' }, `-=${0.3 * 0.6}`);

// === 状态切换 ===
if (isMenuOpen) tl.play();     // 从当前时间正向 → addPause 停下
else tl.reverse();             // 从当前时间反向 → 0
```

**为什么可中断**：GSAP `play()` / `reverse()` 都从 `tl.time()` 当前位置继续，永远不会"跳"。hover in → 立即 pause + reverse；hover out → 立即 play。这就是 demo 标题里"interruptible"的含义。

**注意**：单 timeline 的 `reverse()` 会自动反转 enter 段的 ease（`back.out` → `back.in`，`power2.out` → `power2.in`），这与 demo 中显式 `easeReverse` 的体验略不同。**推荐方案是 B 模式**：把 exit 段独立写出来（上面示例），用完全不同的 `power3.in`，让"撤销"感觉更干脆。

### 2. useGSAP 重建策略

```ts
useGSAP(() => {
  // 重建 timeline
  desktopTlRef.current = gsap.timeline({ paused: true })...;
}, [prefersReducedMotion, isMobile, scene.id]);
```

依赖项选择原则：
- **prefersReducedMotion**：切换时必须重建（动画策略整体不同：reduce 时直接 `gsap.set` 同步样式，不创建 timeline）
- **isMobile**：桌面/移动 timeline 完全不同（width 动画 vs xPercent 抽屉），必须重建
- **scene.id**：场景切换时热点列表 refs 整体更新，timeline 需重建

**重要陷阱**：重建后 useEffect 仍持有旧 ref 的"如果 !tl 跳过"分支，必须保证 useGSAP 同步执行（React 18+ 中 useGSAP ≈ useLayoutEffect）→ useEffect 看到新 ref 之前已经就绪。

### 3. 避免 CSS transition 与 GSAP 冲突

SceneViewer 中曾有：
```tsx
<aside className="... transition-all duration-500 ease-[cubic-bezier(0.25,1,0.5,1)]" style={{ width: 320 }}>
```

**问题**：GSAP `gsap.set(aside, { width: 60 })` 设置内联 style 后，Tailwind `transition-all` 仍会从旧值过渡到新值，**与 GSAP timeline 争夺控制权**。

**解决**：完全移除 `transition-all`，由 GSAP 100% 接管：
```tsx
<aside ref={asideRef} className="... flex flex-col shadow-2xl overflow-hidden">
{/* 不写 width，由 GSAP 写入内联 width；不写 transition，零冲突 */}
```

### 4. pointer-events 同步

`opacity: 0` 元素仍占据布局空间，且**仍接受 pointer-events**。如果折叠态下隐藏了内容区（视觉不可见），用户点击折叠条上方的"看不见"列表项仍会触发选中。

**解决**：在 useEffect 中根据 `isMenuOpen` 同步设置：
```ts
contentRef.current.style.pointerEvents = isMenuOpen ? 'auto' : 'none';
collapsedBarRef.current.style.pointerEvents = (isMobile || isMenuOpen) ? 'none' : 'auto';
```

### 5. backdrop-blur 与 3D 变换冲突

`backdrop-blur-*` 创建独立堆叠上下文，**会强制 `transform-style: flat`**，使子元素的 `preserve-3d` 失效。本项目 Sidebar 使用 `backdrop-blur-xl`，所以**只能使用 2D 变换**（`x` / `xPercent` / `opacity`），不能 `transform: rotateX()` 等 3D 效果。

### 6. 列表 stagger 的两种模式

| 模式 | 效果 | 适用 |
|------|------|------|
| `stagger: 正数` | 顺次错落（1, 2, 3, ...） | enter 段 |
| `stagger: 正数` + `from: 'end'` | 逆序错落 | exit 段（从底部先消失） |

本项目列表 enter 用正向 stagger，exit 仍用正向 stagger（同时淡出 + 微小 x 位移），保留"从下到上消失"的视感。

### 7. useGSAP 多分支初始化的「交叉污染」陷阱

**踩坑场景**：SceneViewer 侧边栏同时支持桌面（hover 展开 60px↔320px）和移动（点击 FAB 抽屉 xPercent 100↔0）两套动画。

**错误实现**：

```ts
useGSAP(() => {
  if (prefersReducedMotion) { ... return; }

  // 桌面端初始
  gsap.set(aside, { width: 60 });
  const desktopTl = gsap.timeline({ paused: true });
  // ... enter/exit ...
  desktopTlRef.current = desktopTl;

  // 移动端初始（**bug：永远会执行**）
  gsap.set(aside, { xPercent: 100, width: '100%' });
  const mobileTl = gsap.timeline({ paused: true });
  // ... enter/exit ...
  mobileTlRef.current = mobileTl;
}, [prefersReducedMotion, isMobile, scene.id]);
```

**症状**：桌面端（`isMobile=false`）时，aside 被设两次：先 `width: 60` 后 `xPercent: 100, width: '100%'`。第二次覆盖第一次，aside 整体被推出视口右侧 100% 自身宽，**完全不可见**。截图上看 sidebar 整块消失。

**根因**：`useGSAP` 内的所有 `gsap.set` / `gsap.timeline` 都是**对真实 DOM 的副作用**，与 `isMobile` 条件无关。把"桌面端和移动端"写成两段顺序代码，会让两段都执行。

**正确实现**：

```ts
useGSAP(() => {
  if (prefersReducedMotion) {
    // reduce 分支：用同一份逻辑同步 set
    gsap.set(aside, { width: collapsedW, xPercent: 0 });
    return;
  }

  // ⚠️ 先把两条 ref 都置空，避免切换时拿到旧 timeline
  desktopTlRef.current = null;
  mobileTlRef.current = null;

  if (isMobile) {
    gsap.set(aside, { width: '100%', xPercent: 100 });
    // ... 构建 mobileTl ...
    mobileTlRef.current = mobileTl;
    return;                                          // 早返
  }

  gsap.set(aside, { width: collapsedW, xPercent: 0 });
  // ... 构建 desktopTl ...
  desktopTlRef.current = desktopTl;
}, [prefersReducedMotion, isMobile, scene.id]);
```

**三条防线**：
1. **`if/else + 早返` 隔离分支**：桌面端不会跑到移动端 `gsap.set`；反之亦然
2. **互斥 ref 显式置空**：切换 `isMobile` 重建时，旧 ref 不残留
3. **依赖项含 `isMobile`**：模式切换自动重建 timeline

**调试技巧**：当 `gsap.set` 出现"看起来没生效"时，先用 DevTools 检查目标元素的 `computed style`：
- `transform: matrix(1, 0, 0, 1, 1280, 0)` → `xPercent: 100` 把元素推走了
- `width: 1280px` 而非期望的 60px → 被后一次 `gsap.set` 覆盖

**预防**：写多模式 `useGSAP` 时，**永远不要**让两套副作用代码段顺序执行；必须用条件分支隔离，并显式 `return`。

### 8. 鼠标驱动 transform 的"三件套"配置

所有"鼠标移动驱动 transform"的动画，都必须同时设三个属性：

```ts
gsap.to(el, {
  rotateX: ..., rotateY: ...,
  transformPerspective: 800,     // 透视距离
  transformOrigin: '50% 50%',   // 旋转原点
  duration: 0.4, ease: 'power2.out',
  overwrite: 'auto',            // 关键：避免 hover/leave 互相覆盖
});
```

**`overwrite: 'auto'` 是关键**：没有它，快速 hover/leave 会让多个 `gsap.to` 互相打架，出现"跳变"。

### 9. backdrop-blur 与 3D 变换的冲突

**现象**：本项目 HomeSelection 顶栏用 `backdrop-blur-xl` 包标题组，标题组内的 `rotateX/rotateY` 看起来像 2D 倾斜，没有透视感。

**根因**：`backdrop-blur-*` 创建独立堆叠上下文，**会强制 `transform-style: flat`**，使子元素的 `preserve-3d` 失效。

**两种解决方案**：

| 方案 | 做法 | 取舍 |
|------|------|------|
| A | 把 `backdrop-blur` 应用在**最外层容器**，让需要 3D 变换的子元素不再被 `transform-style: flat` 约束 | 布局重构成本高 |
| B（推荐） | 放弃 `preserve-3d`，用 2D 视差 + `transformPerspective` 模拟深度 | 改动小；GSAP 的 `transformPerspective` 走独立渲染路径，不受 CSS 堆叠上下文影响 |

本项目顶栏采用方案 B，所有倾斜/视差都由 GSAP 驱动，`transformPerspective` 通过 `gsap.set` / `gsap.to` 写入内联 style，**绕开 CSS 堆叠上下文的限制**。

### 10. 斜切 + 3D 倾斜的"双 skew"技巧

容器 `skewX(-12)` 会让内容**整体跟着倾斜**，看起来"歪"。解决：

```ts
// 容器倾斜
gsap.set(sloganRef.current, {
  skewX: -12,
  transformOrigin: 'bottom right',
  transformPerspective: 600,
});

// 文字反向补偿
gsap.set(sloganTextRef.current, { skewX: 12 });

// 鼠标移动时，必须重设容器和文字的 skewX
gsap.to(sloganRef.current, { skewX: -12, rotateX: ..., rotateY: ..., ... });
gsap.to(sloganTextRef.current, { skewX: 12, x: ..., y: ..., ... });
```

**踩坑**：松开鼠标时 `gsap.to` 不会"保留" `skewX`（如果只设 `rotateX/rotateY: 0`），文字会突然变直。**必须在每次动画里显式带 skewX**。

**进阶**：标语悬停时叠加 `letterSpacing 0.3em → 0.35em` + `textShadow` 暖色阴影，配合弹性归零 `elastic.out(1, 0.5) 0.8s`，可以做出"有手感"的回弹。

### 11. 弹性回弹的"过冲"参数

`elastic.out(amp, period)`：
- `amp`（振幅）越大过冲越多
- `period`（周期）越小回弹越快

本项目悬停归零统一用 `elastic.out(1, 0.5) duration: 0.8`——既不会"软绵绵"也不会"弹太远"。如果元素比较小（如按钮）可降为 `elastic.out(0.8, 0.4) duration: 0.6`；如果元素比较大（如标题）可升至 `elastic.out(1.2, 0.6) duration: 1.0`。

### 12. 悬停加速 timeline.timeScale

用 `gsap.to(continuousTl, { timeScale: 2.5, duration: 0.4 })` 让循环动画**平滑加速**，而不是直接 `timeScale(2.5)` 突变。duration 越小加速越突兀，越大越柔和。

**搭配技巧**：加速时同时给"主光晕"加 `opacity 0.7 → 0.85` 的补强，让"被注视"的感觉更强烈。

### 13. GSAP 轮播无缝循环

**场景**：Hero 区域多张大图自动轮播，最后一张切回首图时无跳变。

**方案**：首图重复追加到末尾（`[...images, images[0]]`），timeline 正常播放到最后一张停留后，`gsap.set` 归零 `yPercent: 0`。因为最后一张和首图是同一张，视觉上无跳变。

```ts
const heroSliderImages = [...HERO_CONFIG.images, HERO_CONFIG.images[0]];
const sliderTl = gsap.timeline({ repeat: -1 });
sliderTl.to({}, { duration: stayDuration }); // 首图停留
for (let i = 1; i < totalImages; i++) {
  sliderTl
    .call(() => playBigTextAnimation()) // 轮播切换时重入场大文本
    .to(sliderContainerRef.current, { yPercent: -i * 100, duration: 1.2, ease: 'power3.inOut' })
    .to({}, { duration: 4 });
}
sliderTl.set(sliderContainerRef.current, { yPercent: 0 }); // 归零循环
```

**注意**：`gsap.set` 放在 timeline 末尾（而非 `repeat` 回调里），确保归零动作在最后一帧停留结束后才执行。

### 14. ScrollTrigger + gsap.context 清理

**场景**：ScrollVideo 组件需要在卸载时清理所有 ScrollTrigger 实例。

**方案**：用 `gsap.context()` 包裹所有动画定义，卸载时调用 `ctx.revert()`：

```ts
const ctx = gsap.context(() => {
  gsap.fromTo(videoEl, { scale: 0.6, opacity: 0.5 }, {
    scale: 1, opacity: 1,
    scrollTrigger: { trigger: videoEl, start: 'top 90%', end: 'top 30%', scrub: 0.5 },
  });
}, containerRef);
return () => ctx.revert();
```

**经验**：`gsap.context()` 是 GSAP 3.x 推荐的清理方式，比手动 `ScrollTrigger.getAll().forEach(t => t.kill())` 更安全——它只清理当前 context 内的动画，不会误杀其他组件的 ScrollTrigger。

### 15. SplitText 字符 stagger 入场

**场景**：Hero 背景大文字 "Environment Art" 需要逐字符入场。

**方案**：`gsap/SplitText` 将文本拆分为单字符 span，配合 stagger 实现错落入场：

```ts
import SplitText from 'gsap/SplitText';
gsap.registerPlugin(SplitText);

const split = SplitText.create(bigTextRef.current, { type: 'chars' });
gsap.from(split.chars, {
  yPercent: 100, opacity: 0, duration: 0.7,
  stagger: 0.04, ease: 'power4.out', delay: 0.3,
});
```

**注意**：SplitText 是 GSAP 付费插件（Club GreenSock），需确认项目 license。字符 DOM 节点在 `split.revert()` 后会被清理，如果需要重复入场动画（如轮播切换时），应在每次动画前重新 `SplitText.create`。

### 16. 点阵噪声 ASCII 字符网格

**场景**：Hero 区域覆盖层需要随机位置显示 ASCII 字符，制造"终端数据流"效果。

**方案**：按网格划分区域，随机选取 `activeRate` 比例的格子填充字符：

```ts
const DOT_ASCII_CHARS = ['.', ':', '-', '0', '4', '6', '8', '9', 'A', '/', '\\', '+'];
const cols = Math.floor(width / DOT_GRID_SIZE);
const rows = Math.floor(height / DOT_GRID_SIZE);
const totalCells = cols * rows;
const activeCount = Math.floor(totalCells * activeRate);

// 随机选取不重复的格子
const indices = new Set<number>();
while (indices.size < activeCount) {
  indices.add(Math.floor(Math.random() * totalCells));
}

// 15% 概率使用强调色，其余白色
for (const idx of indices) {
  const isHighlight = Math.random() < 0.15;
  // 创建 span 元素...
}
```

**性能**：896×411 视口下约 44×20 = 880 格，2% 活跃率仅 17 个 DOM 节点。`setInterval` 180ms 刷新时销毁旧节点、创建新节点，避免大量 DOM 操作。

### 17. playTextScramble 文本乱码解码

**场景**：HUD 文本 hover 时触发"解密"效果，从随机字符逐步过渡到最终文本。

**方案**：`requestAnimationFrame` 逐帧计算 `progress`，每个字符按阈值决定是否揭示：

```ts
function playTextScramble(element: HTMLElement, finalText: string, duration = 520) {
  const chars = '/\\|_+-=<>:;.*#01X';
  let frameId: number;
  const start = performance.now();

  function tick(now: number) {
    const progress = Math.min((now - start) / duration, 1);
    let result = '';
    for (let i = 0; i < finalText.length; i++) {
      if (finalText[i] === ' ') { result += ' '; continue; }
      // 阈值 = 索引占比 + 随机扰动，逐帧不同
      const threshold = (i / finalText.length) + Math.random() * 0.28;
      result += progress > threshold
        ? finalText[i]
        : chars[Math.floor(Math.random() * chars.length)];
    }
    element.textContent = result;
    if (progress < 1) frameId = requestAnimationFrame(tick);
    else element.textContent = finalText;
  }
  frameId = requestAnimationFrame(tick);
  return () => cancelAnimationFrame(frameId);
}
```

**关键**：`random * 0.28` 扰动让每个字符的揭示时机不完全线性，产生"逐字解密"的随机感。`progress >= 1` 时强制设为最终文本，确保动画结束状态正确。

### 18. LERP 鼠标跟随的调参经验

**场景**：HotspotPreview 预览卡片需要跟随鼠标移动，但不能太"硬"也不能太"软"。

**方案**：`requestAnimationFrame` + 线性插值：

```ts
function tick() {
  currentX += (targetX - currentX) * 0.14;
  currentY += (targetY - currentY) * 0.14;
  // 应用位置...
  frameId = requestAnimationFrame(tick);
}
```

**调参经验**：

| LERP 系数 | 感受 | 适用场景 |
|---|---|---|
| 0.05~0.08 | 非常"粘滞"，有延迟感 | 大型元素、背景视差 |
| 0.10~0.15 | 平滑跟随，有"重量感" | 预览卡片、tooltip |
| 0.20~0.30 | 灵敏跟随，几乎无延迟 | 光标、小图标 |
| >0.30 | 太灵敏，失去平滑感 | 不推荐 |

本项目 HotspotPreview 用 0.14，配合视口边界限制（X: [160, vw-120], Y: [220, vh-120]）和速度旋转（`velocityX * 0.08`，限制 ±6°），形成有物理感的跟随效果。

## 移除 Tailwind 主题切换的标准步骤

### 1. 清理链路总览

移除 Tailwind `darkMode: 'class'` 主题切换时，涉及以下 5 步，**顺序不可颠倒**：

| 步骤 | 操作 | 文件 |
|------|------|------|
| ① | 清理所有 `dark:` Tailwind 类（保留暗色值、移除亮色值和 `dark:` 前缀） | 所有含 `dark:` 的组件 |
| ② | 删除切换入口组件（ThemeToggle）及 import | ThemeToggle.tsx + App.tsx |
| ③ | 删除 App.tsx 中的 `isDark` state、`toggleTheme` 函数、`useEffect` 同步逻辑 | App.tsx |
| ④ | 移除 `<html class="dark">`、删除 `darkMode: 'class'`、将 `.dark {}` 变量提升为 `:root {}`、删除 body 主题过渡动画 | index.html |
| ⑤ | 更新文档（UI框架Map、apps-code-map、项目开发经验） | docs/ |

### 2. `dark:` 类的处理策略

当项目从双主题收敛为暗色唯一时，`dark:` 类的处理方式：

- **有亮色和暗色两套值**（如 `opacity-60 dark:opacity-40`）→ 删除亮色值，保留暗色值并移除 `dark:` 前缀 → `opacity-40`
- **仅暗色有值**（如 `dark:bg-background/10`）→ 移除 `dark:` 前缀 → `bg-background/10`
- **暗色为禁用**（如 `dark:invert-0`）→ 直接删除（`invert-0` 等同于无效果，可省略）

### 3. CSS 变量迁移

`index.html` 中 `.dark {}` 块的变量必须提升到 `:root {}`，否则所有 `rgb(var(--background))` 引用将失效（浏览器回退到初始值）：

```css
/* 移除前 */
:root { --background: 245 243 240; }  /* 亮色 */
.dark { --background: 5 5 5; }        /* 暗色 */

/* 移除后 */
:root { --background: 5 5 5; }        /* 暗色唯一 */
```

### 4. 不可遗漏的细节

- `body` 上的 `transition: background-color 0.5s ease, color 0.5s ease` 必须删除（无切换后无意义，且会导致首次加载闪烁）
- Tailwind 配置中的 `darkMode: 'class'` 必须删除，否则 Tailwind 仍会编译 `dark:` 类（虽然不会生效，但增大 CSS 体积）
