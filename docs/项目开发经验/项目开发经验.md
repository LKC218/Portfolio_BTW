# 项目开发经验

## 项目概述

**项目名称**: Portfolio_BTW (作品集_demo)  
**项目类型**: 赛博朋克风格 3D 场景作品集展示  
**技术栈**: React 19 + TypeScript + Vite + Tailwind CSS  
**最后更新**: 2026-05-28

---

## 环境配置经验

### 1. 首次运行配置流程

```bash
# 步骤1: 安装依赖
npm install

# 步骤2: 配置环境变量
# 创建 .env.local 文件，填入 Gemini API Key
GEMINI_API_KEY=your_api_key_here

# 步骤3: 启动开发服务器
npm run dev
# 默认运行在 http://localhost:3000/
```

### 2. 环境变量说明

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `GEMINI_API_KEY` | 否 | Google Gemini API 密钥，用于 AI 分析功能 |

**注意**: 未配置 API Key 时，AI 分析功能会返回模拟数据，不影响其他功能。

## 架构设计经验

### 1. 状态管理模式

项目采用**集中式状态管理**，所有核心状态在 `App.tsx` 中统一管理：

```typescript
// 核心状态
const [appState, setAppState] = useState<AppState>(AppState.HOME);
const [selectedCollection, setSelectedCollection] = useState<Collection | null>(null);
const [selectedScene, setSelectedScene] = useState<Scene | null>(null);
const [isDark, setIsDark] = useState(true);
const [hoveredColor, setHoveredColor] = useState<string | null>(null);
const [activeHotspot, setActiveHotspot] = useState<Hotspot | null>(null);
```

**优点**: 状态流向清晰，调试方便  
**缺点**: 大型项目可能导致不必要的重渲染

### 2. 路由系统设计

使用**状态机模式**替代传统路由：

```typescript
enum AppState {
  HOME,    // 首页 - 作品集选择
  GALLERY, // 画廊 - 场景列表
  VIEWER   // 查看器 - 场景详情
}
```

**转场动画实现**: 3层叠加的 slide 动画，通过 CSS animation 和 setTimeout 配合。

### 3. 数据架构

```
constants.ts
├── COLLECTIONS[] (18个作品集)
└── SCENES_DB{} (每个作品集6个场景)
    └── getScenesForCollection() (数据转换)
```

**热区位置**: 使用 `Math.random()` 生成随机偏移，增加视觉随机性。

## 组件开发经验

### 1. SceneViewer 组件

**核心功能**: 场景查看与热区交互

**关键实现**:
- 热区定位: 百分比定位 + 响应式适配
- 侧边栏: 桌面端 hover 展开，移动端点击展开
- 瞄准器: 根据热区位置动态计算详情卡片位置

**移动端适配**:
```typescript
const [isMobile, setIsMobile] = useState(false);

useEffect(() => {
  const checkMobile = () => {
    const mobile = window.innerWidth < 768;
    setIsMobile(mobile);
  };
  checkMobile();
  window.addEventListener('resize', checkMobile);
  return () => window.removeEventListener('resize', checkMobile);
}, []);
```

### 2. HomeSelection 组件

**核心功能**: 首页展示与作品集选择

**视觉特性**:
- Hero 区域: 视差滚动效果
- 网格布局: 3列自适应
- Marquee 分隔线: 滚动驱动的动画

### 3. GridOverlay 组件

**核心功能**: 背景网格效果，响应环境色变化

**实现要点**:
- 使用 CSS background 实现网格
- 通过 CSS 变量动态调整颜色
- 热区聚焦时网格产生呼吸效果

## 样式系统经验

### 1. Tailwind CSS 配置

项目使用 Tailwind CSS 原子化类名，支持明暗主题：

```html
<div className="bg-background text-foreground dark:bg-surface">
```

### 2. 动画实现

**转场动画**: 3层叠加实现平滑过渡
```css
@keyframes slide-in {
  0% { transform: translateX(100%); }
  100% { transform: translateX(0%); }
}
```

**热区动画**: ping 效果 + 缩放 + 发光
```css
.animate-[ping_2s_cubic-bezier(0,0,0.2,1)_infinite]
```

### 3. 颜色系统

采用赛博朋克风格的霓虹色系，定义在 `ACCENT_COLORS` 常量中。

## AI 集成经验

### 1. Gemini API 调用

```typescript
const ai = new GoogleGenAI({ apiKey: process.env.API_KEY });
const model = 'gemini-2.5-flash-lite-latest';

const prompt = `
  角色扮演：你是一个来自赛博朋克未来的高级视觉分析AI。
  任务：分析一张名为 "${title}" 的图像细节...
`;

const response = await ai.models.generateContent({
  model: model,
  contents: prompt,
});
```

### 2. 错误处理

- API Key 缺失: 返回模拟数据
- 网络错误: 返回错误提示文案
- 所有错误均有 console.error 记录

## 性能优化经验

### 1. 图片加载

- 所有图片使用外部 CDN (picsum.photos)
- 建议后续添加懒加载和占位图

### 2. 动画性能

- 使用 CSS transform 替代 top/left
- 使用 will-change 提示浏览器优化
- 避免在动画中触发重排

### 3. 状态管理

- 避免在渲染函数中创建新对象
- 使用 useCallback/useMemo 优化回调

## 常见问题解决

### 1. 启动报错: "Module not found"

```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

### 2. AI 分析无响应

检查 `.env.local` 文件中的 `GEMINI_API_KEY` 是否正确配置。

### 3. 样式不生效

确认 Tailwind CSS 已正确配置，检查 `tailwind.config.js`。
