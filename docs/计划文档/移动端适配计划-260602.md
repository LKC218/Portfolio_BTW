# 移动端竖版/横版 UI 适配计划

> 日期：2026-06-02
> 测试参考设备：真我 GT8（CSS 411×896 竖版 / 896×411 横版，DPR 3.5）

---

## 1. 问题诊断

### 1.1 HomeSelection — 问题最多

| 问题 | 竖版影响 | 横版影响 | 严重度 |
|------|----------|----------|--------|
| Hero 时钟 `text-[100px]` | 6位时间约600px，**严重溢出411px屏幕** | 同样溢出 | 🔴 |
| Hero 高度 `h-[85vh]` | 85vh≈762px，留给卡片区域极少 | 85vh≈350px，时钟+标语+版本号**挤不下** | 🔴 |
| Hero 底部布局 `flex-col md:flex-row` | 竖版始终flex-col，时钟和标语垂直堆叠溢出 | 横版896>768触发md:flex-row，但高度仅350px | 🟡 |
| 斜切标语 `px-4 md:px-8 py-2 md:py-3` | 间距偏大，挤压其他元素 | 同上 | 🟡 |
| 卡片高度 `h-[400px]` | 400px≈45%视口高度，单张卡片占近半屏 | 横版400px≈97%视口高度，**几乎全屏** | 🔴 |
| 卡片标题 `text-3xl` | 3xl=30px，在411px宽度内换行频繁 | 同上 | 🟡 |
| 滚动字幕 `text-6xl` | 6xl=60px，文字过高 | 同上 | 🟢 |

### 1.2 Gallery — 基本可用，需微调

| 问题 | 竖版影响 | 横版影响 | 严重度 |
|------|----------|----------|--------|
| 卡片 `h-[280px]` 竖版堆叠 | 每张280px，3张=840px，需大量滚动 | 横版触发md:flex-row，但高度仅411px | 🟡 |
| 卡片标题 `text-2xl` | 尚可 | 横版卡片flex-1时标题空间有限 | 🟢 |

### 1.3 SceneViewer — isMobile 判断在横版失效

| 问题 | 竖版影响 | 横版影响 | 严重度 |
|------|----------|----------|--------|
| `isMobile` 仅判断 `width<768` | 竖版411<768 → isMobile=true ✅ | 横版896>768 → isMobile=false ❌ | 🔴 |
| DetailCard 横版走桌面浮动卡片逻辑 | — | 高度仅411px，浮动卡片空间不足 | 🔴 |
| Sidebar 横版走桌面逻辑 | — | 触屏无悬停，60px收起栏无法展开 | 🟡 |
| TopBar 标题 `md:text-4xl` | 竖版text-xl ok | 横版text-4xl可能与返回按钮挤压 | 🟡 |

### 1.4 DetailCard & FullscreenModelViewer — 横版空间不足

| 问题 | 竖版影响 | 横版影响 | 严重度 |
|------|----------|----------|--------|
| 展开态 `flex-col md:flex-row` | ok | 横版md:flex-row触发，但高度411px，3D+面板**严重挤压** | 🔴 |
| FullscreenModelViewer 信息面板 `max-h-[55vh]` | 55vh≈493px ok | 55vh≈226px，**内容放不下** | 🔴 |
| 3D视口 `min-h-[50vh]` | 50vh≈448px ok | 50vh≈206px，**太矮** | 🟡 |

### 1.5 GridOverlay — 需移动端精简

| 问题 | 竖版影响 | 横版影响 | 严重度 |
|------|----------|----------|--------|
| 四角数据 `top-8 left-8` 等 | 可能与顶栏/内容重叠 | 同上 | 🟡 |
| 瞄准环 `w-[40vw]` | 40vw≈164px ok | 40vw≈358px，**过大** | 🟡 |

---

## 2. 核心策略

1. **引入 `useDeviceState` Hook**：同时检测 `width < 768` 和 `orientation`，区分三种状态：`desktop` / `mobile-portrait` / `mobile-landscape`
2. **竖版原则**：内容纵向排列，字号缩小，间距紧凑，避免水平溢出
3. **横版原则**：利用宽度优势横向布局，但高度受限时压缩纵向间距，关键内容优先显示
4. **行距与溢出控制**：所有文本容器加 `overflow-hidden` + `line-clamp`，关键区域用 `max-h` + `overflow-y-auto`

## 3. 实施步骤

### Phase 1: HomeSelection 移动端适配

**1.1 引入方向检测**
- 在组件中添加 `isMobilePortrait` / `isMobileLandscape` 状态
- 基于 `window.innerWidth < 768` + `window.innerHeight > window.innerWidth` 判断

**1.2 Hero 区域竖版适配**
- 时钟：`text-[100px]` → `text-[48px] sm:text-[64px] md:text-[100px] lg:text-[200px]`
- Hero 高度：`h-[85vh]` → `h-[70vh] sm:h-[80vh] md:h-[90vh]`
- 底部内容区：减小 `pb-12` → `pb-6 sm:pb-12`，`gap-8` → `gap-4 sm:gap-8`
- 斜切标语：`text-lg` → `text-sm sm:text-lg`，`px-4 py-2` → `px-3 py-1.5 sm:px-4 sm:py-2`
- 版本号文字：`mt-2` → `mt-1`

**1.3 Hero 区域横版适配**
- Hero 高度：横版时压缩为 `h-[70vh]`
- 底部布局：横版时强制 `flex-row`，时钟和标语左右排列
- 时钟：横版 `text-[80px]`，与标语并排

**1.4 卡片网格适配**
- 竖版卡片高度：`h-[400px]` → `h-[320px] sm:h-[400px]`
- 横版卡片高度：`h-[280px]`
- 卡片标题：`text-3xl` → `text-2xl sm:text-3xl`
- 卡片描述：确保 `line-clamp-2` 生效

**1.5 滚动字幕适配**
- 竖版：`text-4xl sm:text-6xl`
- 横版：`text-4xl`

---

### Phase 2: Gallery 移动端适配

**2.1 竖版微调**
- 卡片高度：`h-[280px]` → `h-[240px] sm:h-[280px]`
- 卡片标题：`text-2xl` → `text-xl sm:text-2xl`
- 确保描述文字 `line-clamp-2` 在小屏幕生效

**2.2 横版专项**
- 横版宽度896>768，已触发 `md:flex-row`，卡片横向排列 ✅
- Header 高度可压缩：`h-16 md:h-20`

---

### Phase 3: SceneViewer 移动端适配

**3.1 修复 isMobile 判断逻辑**
- 当前：`window.innerWidth < 768`
- 修改为：`window.innerWidth < 768 || (window.innerWidth < 1024 && window.innerHeight < 500)`
- 这样横版手机 (896×411) 也会被识别为移动端

**3.2 横版移动端 DetailCard 定位**
- 横版 isMobile=true 时，DetailCard 不再用底部抽屉
- 改为右侧浮动卡片，`max-h-[80vh]`

**3.3 横版移动端 Sidebar**
- 横版 isMobile=true 时，仍用抽屉式从右侧滑入
- 抽屉宽度改为 `max-w-[60vw]` 而非 `w-full`

**3.4 TopBar 横版适配**
- 标题：横版移动端用 `text-xl` 而非 `text-4xl`
- 副标题：确保不换行溢出

---

### Phase 4: DetailCard & FullscreenModelViewer 移动端适配

**4.1 DetailCard 横版适配**
- 新增 `isLandscape` 判断
- 横版移动端：右侧浮动面板 `w-[60vw] max-h-[85vh]`
- 展开态横版：3D视口和信息面板左右排列，信息面板 `w-72`
- 信息面板内容：压缩间距，标题 `text-xl`

**4.2 FullscreenModelViewer 横版适配**
- 横版移动端：3D视口占60%，信息面板占40%
- 信息面板 `max-h-none`，改为左右布局
- 操作指南：横版时更紧凑

---

### Phase 5: GridOverlay & 全局 UI 移动端适配

**5.1 GridOverlay 移动端精简**
- 四角数据：移动端隐藏或缩小字号
- 瞄准环：横版 `w-[30vw] h-[30vw]`
- 光斑效果：移动端降低 blur 值以提升性能

**5.2 AudioPlayer / ThemeToggle 移动端**
- 确保不与底部内容重叠
- 横版时位置微调

---

### Phase 6: 测试验证

**6.1 Chrome DevTools 模拟测试**
- 竖版：真我 GT8 (411×896, DPR 3.5)
- 横版：真我 GT8 横版 (896×411, DPR 3.5)
- iPhone SE (375×667)
- iPad Mini (768×1024)

**6.2 验证清单**
- [ ] 所有页面无水平溢出（无横向滚动条）
- [ ] 文字行距合理，无重叠
- [ ] 交互元素（按钮、卡片）可正常点击
- [ ] 底部抽屉/侧边栏在横版正常工作
- [ ] 3D 查看器在横版有足够视口空间
- [ ] 全局 UI 不遮挡关键内容
- [ ] `prefers-reduced-motion` 仍然生效
